01 · Background & Research Gap

Research Domain

Research Background

Tamil Sign Language is an important communication method for hearing-impaired people in Sri Lanka. Existing manual interpretation and static sign recognition systems are limited, so a real-time video-based recognition system is needed.

Research Gap & Limitations

Most existing studies focus on static signs, controlled environments, or other sign languages like ASL and ISL. Very few works handle dynamic Tamil Sign Language using real-time webcam/video input with lightweight models for accessibility.

02 · System Limitations

Existing challenges

Tamil Sign Language support systems face critical gaps that limit accessibility, accuracy, and real-world deployment for hearing-impaired communities.

GAP 01

Limited TSL resources

Existing sign language datasets are dominated by ASL and BSL, leaving Tamil Sign Language with little to no annotated corpora for training reliable models.

GAP 02

No multimedia conversion

Current tools lack the ability to convert spoken Tamil audio or video content into TSL, cutting off hearing-impaired users from everyday media and education.

GAP 03

Hardware dependency

Many gesture recognition systems rely on depth cameras, glove sensors, or multi-camera rigs - raising cost and blocking adoption in low-resource environments.

GAP 04

Static over dynamic signs

Most systems classify isolated hand poses and ignore the fluid, transitional movements that are central to real TSL communication sequences.

GAP 05

No learning platform

There is no dedicated interactive platform for learning TSL with structured evaluation, progress tracking, or mentor oversight tailored to Tamil speakers.

GAP 06

Opaque model predictions

Deep learning models used in sign detection often act as black boxes, eroding trust among educators and making it difficult to diagnose or improve accuracy.

03 · Research Problem

Communication barriers

Hearing-impaired communities depend on accessible technology—yet affordable, real-time recognition remains scarce.

The Core Challenge

Approximately 300,000 individuals in Sri Lanka rely on Tamil Sign Language as their primary means of communication. Yet assistive technology that interprets signing from ordinary cameras remains scarce, fragmented, or priced out of reach.

The lack of standardised lexicons, the subtle fluidity of transitional movements, and the diversity of signing environments make robust recognition a genuinely hard problem — one that must be solved transparently and affordably to matter at community scale.

CORE QUESTION

Five research questions

  • Can holistic landmarks alone discriminate between 15 TSL glosses?
  • Does a fixed-length window keep training and live inference comparable?
  • Can the entire stack run on consumer hardware?
  • How much does Bi-GRU trade off against Bi-LSTM?
  • How can explainable AI increase user trust?
04 · Research Objectives

What this project achieves

A deployable, transparent, and affordable pipeline for dynamic Tamil Sign Language recognition.

PRIMARY OBJECTIVE

Real-time TSL Recognition

Deploy a transparent system that converts dynamic TSL gestures into text with sufficient accuracy for schools, homes, and community centres.

OBJ-01

Gesture Accuracy

Use holistic landmarks and PCA to exceed practical deployment thresholds.

OBJ-02

Low Latency

Keep inference fast on consumer laptops without GPUs for real-time conversation.

OBJ-03

Continuous Detection

Design sliding buffers for natural progression to continuous recognition.

OBJ-04

Explainability

Surface confidence scores and feature attributions for user trust.

05 · System Workflow

Recognition pipeline

From video input to real-time Tamil Sign Language predictions.

1
Input
Video input

User uploads a pre-recorded sign video or opens the webcam for live gesture capture. Both modes feed into the same processing pipeline.

Webcam Video upload React UI
2
Extraction
Landmark extraction

MediaPipe Holistic processes each frame to extract 33 pose, 468 face, and 42 hand landmarks - no depth sensors or gloves required.

MediaPipe Holistic OpenCV
3
Preprocessing
Normalise and reduce

Raw landmark data is normalised, standardised, and dimensionality-reduced using PCA. Scaler and PCA rules are serialised and shared between training and inference to ensure consistency.

PCA NumPy Scikit-learn
4
Prediction
Model classification

The preprocessed sequence is passed to a Bi-LSTM or Bi-GRU model which classifies the sign gesture across a 45-frame temporal window and returns a prediction with a confidence score.

Bi-LSTM Bi-GRU TensorFlow / Keras Flask API
5
Output
Output and explainability

The predicted TSL gloss, a 0-100 recognition confidence score, and SHAP-based feature importance attribution are displayed to the user for transparency and trust.

SHAP Confidence scoring React UI
